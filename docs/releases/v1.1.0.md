# v1.1.0

## 本次发布

这是一次发布流程整理，不改变 MeTube 服务器功能，也不改变下载目录和下载逻辑。

## 新增

- 增加 `CHANGELOG.md`，以后所有正式版本都要写变更日志。
- 增加 `docs/release-process.md`，规定什么时候发 Release、发布前检查什么、怎么回滚。
- 增加 GitHub Actions 质量门禁，自动检查脚本语法和敏感信息。
- 增加 Pull Request 模板，要求写清变更、验证和安全检查。
- 增加 GitHub Release Notes 分类配置。

## 安全

- 发布检查里明确禁止把服务器地址、密码、Cookie、API Key、SSH Key、安装日记放进公开仓库。
- 发布流程要求先确认仓库公开/私有，再发布。

## 验证

- `node --check metube-zh-cn.js`
- `python -m py_compile ytdl.py subscriptions.py dl_formats.py`
- `git diff --check`
- 本地敏感信息扫描
- GitHub 仓库敏感信息扫描

## 升级说明

这次只是仓库治理和发布流程，不需要重装服务器，也不需要重启 MeTube。

## 回滚说明

如果不想保留这套发布流程，可以回滚本次提交里的文档和 GitHub Actions 文件；服务器不受影响。
